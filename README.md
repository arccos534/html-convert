# HTML Convert

Визуальный конструктор новостей для старого Bitrix с экспортом в безопасный HTML-фрагмент.

Проект позволяет собрать новость из блоков, посмотреть её в режиме редактора и в режиме предпросмотра, а затем вставить готовый HTML прямо в Bitrix через режим редактирования кода.

## Что умеет

- Собирать новость из визуальных блоков
- Перетаскивать блоки drag-and-drop
- Редактировать контент прямо в блоках
- Использовать rich-text форматирование через меню `...`
- Смотреть ленту и страницу новости в `Предпросмотре`
- Экспортировать HTML, безопасный для вставки в Bitrix
- Встраивать изображения прямо в HTML как `data:image`
- Импортировать данные диаграммы/таблицы из `xlsx/xls/csv`

## Bitrix-safe экспорт

Экспорт формирует не целую страницу, а HTML-фрагмент для тела новости:

- весь контент обёрнут в собственный контейнер
- стили заскоуплены внутрь этого контейнера
- нет глобального влияния на шаблон сайта
- загруженные изображения могут быть встроены прямо в код

Это нужно для сценария, когда новость вставляется в Bitrix не через визуальный редактор, а через HTML-код.

## Изображения

Блок изображения поддерживает два режима:

- загрузка файла изображения
- вставка URL с последующим встраиванием в код

Если изображение загружено файлом, оно автоматически превращается в `data:image...` и уходит в HTML без зависимости от внешнего сервера.

Также у блока изображения есть:

- масштабирование
- выравнивание `слева / по центру / справа`
- текст рядом с картинкой
- выбор стороны `картинка слева` или `картинка справа`

## Диаграммы и таблицы

Блок диаграммы умеет:

- принимать `xlsx`, `xls`, `csv`
- автоматически распознавать, что в файле: диаграммные данные или таблица
- превращать импортированный файл в аккуратный сайтный вид
- сохранять это в экспортируемом HTML

Для изображений диаграмм также можно загружать готовые файлы напрямую.

## Запуск

Установка зависимостей:

```bash
npm install
```

Режим разработки:

```bash
npm run dev
```

Продакшен-сборка:

```bash
npm run build
```

Локальный просмотр production-сборки:

```bash
npm run preview
```

## Структура проекта

- `src/App.tsx` — основная логика редактора
- `src/components/editor` — верхняя панель, библиотека блоков, canvas, настройки, preview
- `src/components/blocks` — рендер и inline-редактирование блоков
- `src/data/blockFactory.ts` — фабрики блоков и шаблоны
- `src/lib/exportHtml.ts` — генерация итогового HTML
- `src/lib/sitePreview.ts` — сайтоподобный preview новости
- `src/lib/chartImport.ts` — импорт таблиц и диаграмм из файлов
- `src/lib/documentIO.ts` — работа с файлами, clipboard и встраиванием изображений
- `src/types.ts` — типы документа и блоков

## Текущий статус

Проект находится в активной доработке под конкретный редакционный сценарий:

- улучшение блоков по одному
- доведение preview до вида реального сайта
- настройка экспорта под ограничения старого Bitrix

В production build есть предупреждение Vite о размере JS-бандла. Для текущего этапа это ожидаемо: основную массу дают TipTap, drag-and-drop и логика импорта/preview.
